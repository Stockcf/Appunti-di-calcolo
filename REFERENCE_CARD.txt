SCENARIO 1: After Each Lecture
  1. Lecture ends
  2. Add topic to registro.txt (if not already there)
  3. Run: double-click track_all.bat
  4. Review dashboard
  5. See which topics need writing

SCENARIO 2: Writing New Content
  1. Update chapter files in Chapters/
  2. Add new \section{} headers with technical terms
  3. Save files
  4. Run: double-click track_all.bat
  5. Refresh browser (F5) on dashboard
  6. Watch coverage percentage increase

SCENARIO 3: Weekly Review
  1. Check dashboard for pending topics
  2. Prioritize: pending > partial > covered
  3. Plan writing for next week
  4. Document progress
  5. Run tracker again next week to verify

SCENARIO 4: Before Submission
  1. Run tracker: double-click track_all.bat
  2. Check: All lectures at least ◐ PARTIAL?
  3. If any ○ PENDING: add basic content
  4. If any ◐ PARTIAL: expand to full coverage
  5. Final run to verify all ✓ COVERED (or close)
  6. Save topic_status.json as proof of progress

┌────────────────────────────────────────────────────────────────┐
│ CUSTOMIZATION (Advanced)                                       │
└────────────────────────────────────────────────────────────────┘

Change Coverage Threshold:
  In: topic_tracker.py
  Find: if len(covered_keywords) > len(topic_keywords) * 0.7:
  Change: 0.7 to desired value (0.5 = stricter, 0.9 = lenient)

Add More Stopwords:
  In: topic_tracker.py → extract_keywords()
  Edit: stopwords = {'di', 'del', 'della', ...}

Modify Output:
  Edit: print_summary() for console changes
  Edit: generate_html_dashboard() for HTML changes

┌────────────────────────────────────────────────────────────────┐
│ TROUBLESHOOTING                                                │
└────────────────────────────────────────────────────────────────┘

Problem: Batch file closes immediately
Solution: Open Command Prompt, cd to folder, type: python topic_tracker.py
          This shows the actual error message

Problem: "Python is not recognized"
Solution: Install Python from python.org
          During install: CHECK "Add Python to PATH"

Problem: Empty results / No topics found
Solution: Check that registro.txt has lectures in correct format
          Check that Chapters/*.tex files have \section{} headers

Problem: Dashboard doesn't open
Solution: Try double-clicking dashboard.html manually
          Check that topic_status.json exists in the folder

Problem: Incorrect coverage detection
Solution: The system matches keywords exactly
          If topics use different terms, it may miss matches
          Check topic_status.json to verify what was detected
