  "Qual è un modo intelligente di tenere traccia degli argomenti che sono 
   già stati aggiunti agli appunti e quelli che sono sempre da trattare?"

OUR SOLUTION:
  An automated system that intelligently tracks which lecture topics from 
  your registro.txt have been covered in your written chapters (Chapters/) 
  and which still need to be addressed.

THE RESULT:
  ✓ COVERED topics (written and in your notes)
  ◐ PARTIAL topics (started but need expansion)  
  ○ PENDING topics (not yet written)

STATUS INDICATORS
═════════════════════════════════════════════════════════════════════════════

✓ COVERED (Green)
  Meaning: Topic has >70% coverage in your chapters
  Action: Done! Move to next topic
  Example: ✓ Lezione 3 (Mer 18/03/2026) - LUCA HELTAI

◐ PARTIAL (Orange)  
  Meaning: Topic has 10-70% coverage (started but incomplete)
  Action: Expand this topic to full coverage
  Example: ◐ Lezione 2 (Mer 11/03/2026) - LUCA HELTAI

○ PENDING (Red)
  Meaning: Topic has <10% or no coverage (not yet written)
  Action: Highest priority - write this next
  Example: ○ Lezione 4 (Mer 25/03/2026) - LUCA HELTAI

YOUR WORKFLOW
═════════════════════════════════════════════════════════════════════════════

AFTER EACH LECTURE:
  1. Lecture ends
  2. Check: Is it in registro.txt? (Usually yes)
  3. Run: Double-click track_all.bat
  4. Review: Dashboard shows current status
  5. Plan: Identify topics that need writing

WHEN WRITING NEW CONTENT:
  1. Open chapter file (e.g., Chapters/ch01/chapter01.tex)
  2. Add section headers with keywords: \section{Topic Name}
  3. Write content under the section
  4. Save file
  5. Run: Double-click track_all.bat
  6. Verify: Dashboard shows increased coverage ✓

WEEKLY REVIEW:
  1. Run tracker: Double-click track_all.bat
  2. Check dashboard for pending topics
  3. Prioritize: Pending > Partial > Covered
  4. Plan next week's writing based on pending items
  5. Re-run at week end to verify progress

BEFORE SUBMISSION:
  1. Run tracker
  2. Ensure all lectures have at least ◐ PARTIAL coverage
  3. Expand any ◐ PARTIAL to ✓ COVERED if possible
  4. Save topic_status.json as proof of progress
  5. Final verification run
